На основе получаемого от пользователя списка атрибутов сгенерируй структуру JSON согласно следующим правилам:

Входные данные: список имен атрибутов.

Требуемый выходной формат (массив объектов JSON):
[
  {
    "Идентификатор": "названиеАтрибута",
    "Варианты": [
      {
        "Тип": "Справочник",
        "ЕдинственноеЧисло": "Вариант",
        "МножественноеЧисло": "Варианты"
      },
      {
        "Тип": "Справочник",
        "ЕдинственноеЧисло": "ДругойВариант",
        "МножественноеЧисло": "ДругиеВарианты"
      },
      {
        "Тип": "Число",
        "ДлинаЧисла": 10,
        "ТочностьЧисла": 2
      },
      {
        "Тип": "Строка",
        "ДлинаСтроки": 10
      },
      {
        "Тип": "Дата",
        "ЧастиДаты": "Дата"
      }
    ]
  },
  ...
]

Правила заполнения. 
Варианты имен должны быть предложены в порядке предпочтения (до 5 вариантов), учитывая семантику исходного атрибута. Если по имени аттрибута подразуемевается значение простого типа, он должен быть раньше. Например для "Количество", раньше должно идти Число, а затем уже ссылочные типы.
Например, для поля с именем "Год", сперва должно быть Число, затем уже справочник "Годы".
Для свойства "Тип" допустимы только следующие значения:
   - "Число" (если значение подразумевает числовое значение, например: Количество, Сумма, Цена)
   - "Строка" (если значение подразумевает строковое значение, например: Описание, Наименование)
   - "Дата" (если значение подразумевает дату, например ДатаСоздания, ДатаЗаявки)
   - "Булево" (если значение подразумевает да или нет)
   - "Справочник" (основной вариант для ссылочного значения)
   - "Перечисление" (если значение подразумевает ограниченное количество вариантов)
   - "Документ" (если значение подразумевает номер и дату, и для значений важна последовательность)

Дальнейшие значения зависят от типа.
- Для типов "Справочник" и "Перечисление", "Документ" заполни свойства "ЕдинственноеЧисло" и "МножественноеЧисло". 
Если значение не имеет множественного числа - укажи в поле "МножественноеЧисло" форму единственного числа. Например, для "Номенклатура", форма множественного числа будет "Номенклатура"
- Для типа Число укажи ДлинаЧисла и ТочностьЧисла (например, 10 и 2 для денежных значений).
- Для типа Строка укажи ДлинаСтроки.
- Для типа Дата укажи ЧастиДаты (возможные значения: Дата, Время или ДатаВремя).    
- Для типа Булево отсутсвуют дополнительные свойства.

Сгенерируй JSON-структуру для каждого из входящих атрибутов, следуя этим правилам.
